# 📘 GUIDE.md - 知识库使用指南

## 目录结构 (PARA 方法)

```
00_Inbox/       - 收件箱（临时文件，新想法收集）
01_Projects/    - 项目（有明确目标和时间范围）
02_Areas/       - 领域/责任区（长期维护的职责）
03_Resources/   - 资源（参考材料、工具、教程）
04_Archive/     - 归档（已完成/过时的内容）
05_Attachments/ - 附件（图片、文件等）
06_Canvas/      - 画布（脑图、思维导图）
07_Templates/   - 模板
```

## 使用规则

1. **文档分类**：新文档必须放入对应目录，不要放根目录
2. **命名规范**：使用英文或拼音命名，避免中文文件名
3. **每日笔记**：放在 `00_Inbox/` 或按日期放在对应 Area
4. **项目文档**：放在 `01_Projects/`
5. **长期知识**：放在 `02_Areas/`

## 与 AI 协作

- AI 生成的文档默认- 每次修改保存到此仓库
后需要 `git push`
